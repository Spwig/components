🌏 AirWallex Payment Setup

Why Choose AirWallex? Process payments globally with 160+ payment methods across 130+ countries. Competitive FX rates and low transaction fees make AirWallex ideal for international commerce.

Step 1: Create Your AirWallex Account

  1. Visit www.airwallex.com
  2. Click "Sign Up" or "Get Started"
  3. Complete the registration process with your business details
  4. Verify your email address and complete KYC requirements
💡 Tip: Have your business registration documents ready for faster verification.

Step 2: Access Your API Credentials

  1. Log into your AirWallex Dashboard
  2. Navigate to Settings → Developer → API Keys
  3. You'll find your Client ID displayed on this page
  4. Click "Create API Key" to generate a new API key
  5. Choose the appropriate permissions:
    • ✅ Payments - Required for processing transactions
    • ✅ Refunds - Required for refund processing
    • ✅ Read Account - For connection testing
  6. Important: Copy your API Key immediately - it won't be shown again!
⚠️ Security: Never share your API credentials. Keep them secure and don't commit them to version control.

Step 2a: Understanding Demo vs Production Credentials

You will need to provide both demo (sandbox) and production credentials:

  • Demo Credentials: For testing without real charges
    • Use these to test your integration with AirWallex's test card numbers
    • No real money is charged
    • Available in your AirWallex demo environment
  • Production Credentials: For live transactions
    • Use these to process real payments from customers
    • Real money is charged to real payment methods
    • Requires completing KYC verification
💡 Getting Started: You can initially set up with only demo credentials and add production credentials later when you're ready to go live. The wizard will collect both sets of keys, allowing you to easily switch between demo and production mode.
⚠️ Never Mix Keys! Always keep demo and production credentials separate. Using the wrong credentials can result in test transactions appearing in production reports or real charges being processed in demo mode.

Step 3: Configure Webhook (Optional but Recommended)

  1. In your AirWallex Dashboard, go to Developer → Webhooks
  2. Click "Add Webhook"
  3. Enter your webhook URL:
  4. Select the following events:
    • ✅ payment_intent.succeeded
    • ✅ payment_intent.failed
    • ✅ payment_intent.requires_capture
    • ✅ refund.received
    • ✅ refund.accepted
    • ✅ refund.settled
    • ✅ refund.failed
  5. Enable "Signing Secret" and copy the secret key
  6. Click "Save"
💡 Why Webhooks? Webhooks provide real-time payment notifications, ensuring your orders are updated immediately when payments succeed or fail.

Step 4: Enter Your Credentials Below

  1. Demo Credentials: Enter your demo/sandbox keys
    • Demo Client ID
    • Demo API Key
    • Demo Webhook Secret (if configured)
  2. Production Credentials (optional initially): Enter your live keys
    • Production Client ID
    • Production API Key
    • Production Webhook Secret (if configured)
  3. Use the Test Mode toggle to switch between demo and production:
    • Test Mode ON: For development and testing with test payments
    • Test Mode OFF: For real transactions with actual money
  4. Configure your payment preferences (auto-capture, payment descriptor, etc.)
  5. Click "Test Connection" to verify everything works
Tip: Start with Test Mode enabled. You can add production credentials and switch to production mode later when you're ready to go live.

Step 5: Test Your Integration

✅ Testing Mode: When using Demo/Sandbox environment, you can use AirWallex's test card numbers to simulate payments without real money.

Test Cards:

  • Success: 4012 0000 3333 0026 (any future expiry, any CVV)
  • 3D Secure: 4000 0027 6000 3184 (triggers 3D Secure flow)
  • Declined: 4000 0000 0000 0002 (simulates declined payment)

Visit AirWallex Documentation for more test scenarios.

Step 6: Go Live

  1. Complete all AirWallex KYC/verification requirements
  2. Test thoroughly in Demo/Sandbox mode first
  3. When ready, switch Environment to Production
  4. Generate new production API credentials (don't reuse sandbox credentials)
  5. Update your configuration with production credentials
  6. Process a small test transaction to verify
  7. You're ready to accept live payments! 🎉
