Square Payment Setup

Why Choose Square? Accept payments online and in-person with one unified platform. Square supports credit/debit cards, Apple Pay, Google Pay, Cash App Pay, and Afterpay/Clearpay with transparent, competitive pricing and no monthly fees.

Step 1: Create Your Square Account

  1. Visit squareup.com/signup
  2. Click "Get Started" and follow the registration process
  3. Complete your business profile and identity verification
  4. Verify your email address
Tip: Square offers a free account with no monthly fees. You only pay per-transaction processing fees.

Step 2: Create a Square Application

  1. Go to the Square Developer Dashboard
  2. Click the "+" button to create a new application
  3. Give your application a name (e.g., "My Online Store")
  4. Once created, navigate to the "Credentials" tab
  5. Copy your Application ID (starts with sq0idp- for sandbox)
  6. Under "Access token", copy your Access Token
Security: Keep your Access Token secure. Never share it publicly or commit it to version control. Treat it like a password.

Step 3: Get Your Location ID

  1. In the Square Developer Dashboard, go to your application
  2. Navigate to the "Locations" section
  3. Copy the Location ID for the location you want to process payments for
  4. Alternatively, find your Location ID in the Square Dashboard under Account & Settings > Locations
Tip: Each Square account has at least one location. For online-only stores, use your default location.

Step 3a: Understanding Sandbox vs Production Credentials

You will need to provide both sandbox and production credentials:

  • Sandbox Credentials: For testing without real charges
    • Use these to test your integration with Square's test payment nonces
    • No real money is charged
    • Available in your application's Credentials tab (Sandbox mode toggle)
    • Application IDs start with sandbox-sq0idp-
  • Production Credentials: For live transactions
    • Use these to process real payments from customers
    • Real money is charged to real payment methods
    • Requires completing account verification
    • Application IDs start with sq0idp- (no sandbox prefix)
💡 Getting Started: You can initially set up with only sandbox credentials and add production credentials later when you're ready to go live. The wizard will collect both sets of keys, allowing you to easily switch between sandbox and production mode.
⚠️ Never Mix Keys! Always keep sandbox and production credentials separate. Using the wrong credentials can result in test transactions appearing in production reports or real charges being processed in sandbox mode.

Step 4: Configure Webhooks (Recommended)

  1. In the Square Developer Dashboard, go to your application
  2. Navigate to the "Webhooks" tab
  3. Click "Add Endpoint"
  4. Enter your webhook URL:
  5. Select the following events:
    • payment.completed
    • payment.updated
    • refund.created
    • refund.updated
  6. Copy the Signature Key displayed after saving
  7. Click "Save"
Why Webhooks? Webhooks provide real-time payment notifications, ensuring your orders are updated immediately when payments succeed, fail, or are refunded.

Step 5: Test with Sandbox

Testing Mode: Use Sandbox environment credentials to test payments without processing real money. Square provides test payment tokens for this purpose.

Sandbox Test Values:

  • Success nonce: cnon:card-nonce-ok - Simulates a successful card payment
  • Declined nonce: cnon:card-nonce-declined - Simulates a declined card
  • CVV failure: cnon:card-nonce-rejected-cvv-failure - Simulates CVV check failure
  • Address mismatch: cnon:card-nonce-rejected-postalcode-failure - Simulates address verification failure
  • Expiry failure: cnon:card-nonce-rejected-expiration-failure - Simulates expired card

Visit Square Test Values Documentation for more test scenarios.

Step 6: Go Live with Production

  1. Complete all Square account verification requirements
  2. Thoroughly test all payment flows in Sandbox mode
  3. When ready, switch Environment to Production
  4. Use your Production Access Token and Application ID (found in the Credentials tab with "Production" selected)
  5. Update the Location ID if it differs between sandbox and production
  6. Process a small live test transaction to verify everything works
  7. You are ready to accept live payments!
