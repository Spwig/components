Adyen Payment Setup

Why Choose Adyen? Enterprise-grade payment processing with 250+ payment methods across 40+ countries. Industry-leading authorization rates, unified commerce capabilities, and comprehensive fraud protection powered by RevenueProtect.

Step 1: Create Your Adyen Account

  1. Visit adyen.com/signup
  2. Click "Get started" and complete the registration
  3. You will receive access to the Adyen Customer Area (test environment)
  4. Complete your business verification for live processing
Tip: Adyen provides a test environment immediately upon signup. You can start integrating right away while your live account is being verified.

Step 2: Get Your API Key

  1. Log into your Adyen Customer Area
  2. Navigate to Developers → API credentials
  3. Click on your API credential (or create a new one)
  4. Under Authentication, click "Generate API key"
  5. Copy and securely store the API key
Security: The API key is only shown once. Copy it immediately and store it securely. Never share it or commit it to version control.

Step 3: Get Your Client Key

  1. On the same API credential page, scroll to Client settings
  2. Copy your Client key (starts with test_ or live_)
  3. Add your website domain under Allowed origins
  4. The client key is used for the frontend Drop-in/Components integration
Tip: For testing, add http://localhost:8000 to allowed origins. For production, add your actual domain.

Step 4: Configure Webhooks

  1. In the Customer Area, go to Developers → Webhooks
  2. Click "+ Webhook" and select "Standard webhook"
  3. Enter your webhook URL:
  4. Under Additional settings, enable HMAC verification
  5. Copy the generated HMAC key and save it
  6. Select the following events:
    • AUTHORISATION
    • CAPTURE
    • CANCELLATION
    • REFUND
    • REFUND_FAILED
  7. Click "Save configuration"
Why Webhooks? Adyen delivers payment results asynchronously via webhooks. This ensures reliable payment status updates even when customers close their browser during 3DS authentication.

Step 5: Live Environment Setup

  1. Once your live account is approved, go to Settings → API URLs
  2. Copy your Live endpoint prefix (e.g., 1797a841fbb37ca7-YourCompany)
  3. This prefix is required for live API calls
  4. Generate new live API credentials (do not reuse test credentials)
  5. Update your configuration with the live endpoint prefix and new credentials
Important: The live endpoint prefix is unique to your account. Without it, live API calls will fail. This is not needed for the test environment.

Step 6: Test Your Integration

Testing Mode: When using the Test environment, you can use Adyen's test card numbers to simulate payments without real money.

Test Cards:

  • Visa (Success): 4111 1111 1111 1111 (any future expiry, any 3-digit CVC)
  • Visa (Refused): 4000 0000 0000 0002 (simulates refused payment)
  • Mastercard: 5500 0000 0000 0004 (any future expiry, any 3-digit CVC)
  • 3D Secure: 4212 3456 7891 0006 (triggers 3DS authentication)
  • iDEAL: Select iDEAL as payment method in the Drop-in, choose a test bank

Visit Adyen Test Card Numbers for the complete list of test scenarios.
