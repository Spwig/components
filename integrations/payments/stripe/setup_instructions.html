Stripe Payment Setup

Why Choose Stripe? The most popular payment platform for internet businesses. Accept cards, digital wallets, BNPL, and local payment methods in 135+ currencies with industry-leading developer tools and fraud prevention.

Step 1: Create Your Stripe Account

  1. Visit dashboard.stripe.com/register
  2. Enter your email address and create a password
  3. Complete the registration process with your business details
  4. Verify your email address
Tip: You can start testing immediately in test mode, even before completing business verification.

Step 2: Get Your API Keys

  1. Log into your Stripe Dashboard
  2. Navigate to Developers → API keys
  3. You will see two keys:
    • Publishable key (starts with pk_test_ or pk_live_) — used on the frontend
    • Secret key (starts with sk_test_ or sk_live_) — used on the backend
  4. Click "Reveal test key" to copy your secret key
  5. Important: Copy both keys and store them securely
Security: Never share your Secret Key. Keep it secure and do not commit it to version control. The Publishable Key is safe to expose in frontend code.

Step 2a: Understanding Test vs Production Credentials

You will need to provide both test and production credentials:

  • Test Credentials: For testing without real charges (pk_test_, sk_test_)
    • Use these to test your integration with Stripe's test card numbers
    • No real money is charged
    • Available immediately after creating your account
  • Production Credentials: For live transactions (pk_live_, sk_live_)
    • Use these to process real payments from customers
    • Real money is charged to real credit cards
    • Requires completing business verification
💡 Getting Started: You can initially set up with only test credentials and add production credentials later when you're ready to go live. The wizard will collect both sets of keys, allowing you to easily switch between test and production mode.
⚠️ Never Mix Keys! Always keep test and production keys separate. Using the wrong key can result in test charges appearing in production reports or real charges being processed in test mode.

Step 3: Configure Webhook (Recommended)

  1. In your Stripe Dashboard, go to Developers → Webhooks
  2. Click "Add endpoint"
  3. Enter your webhook URL:
  4. Select the following events to listen for:
    • payment_intent.succeeded
    • payment_intent.payment_failed
    • charge.refunded
    • charge.refund.updated
    • checkout.session.completed
  5. Click "Add endpoint" to save
  6. After creating the endpoint, click on it and copy the Signing secret (starts with whsec_)
Why Webhooks? Webhooks provide real-time payment notifications, ensuring your orders are updated immediately when payments succeed, fail, or are refunded.

Step 4: Enter Your Credentials Below

  1. Test Credentials: Paste your test keys
    • Test Publishable Key (pk_test_...)
    • Test Secret Key (sk_test_...)
    • Test Webhook Secret (whsec_...) if you configured a test webhook
  2. Production Credentials (optional initially): Paste your live keys
    • Live Publishable Key (pk_live_...)
    • Live Secret Key (sk_live_...)
    • Live Webhook Secret (whsec_...) if you configured a live webhook
  3. Use the Test Mode toggle to switch between test and production:
    • Test Mode ON: For development and testing with test card numbers
    • Test Mode OFF: For real transactions with actual money
  4. Click "Test Connection" to verify everything works
Tip: Start with Test Mode enabled. You can add production credentials and switch to production mode later when you're ready to go live.

Step 5: Test Your Integration

Testing Mode: When using Test environment, use Stripe's test card numbers to simulate payments without real money.

Test Cards:

  • Success: 4242 4242 4242 4242 (any future expiry, any 3-digit CVC)
  • 3D Secure: 4000 0025 0000 3155 (triggers 3DS authentication)
  • Declined: 4000 0000 0000 0002 (simulates declined payment)
  • Insufficient funds: 4000 0000 0000 9995

Visit Stripe Testing Documentation for more test scenarios and international test cards.

Step 6: Go Live

  1. Complete your Stripe account activation (business verification, bank account, etc.)
  2. Test thoroughly in Test mode first
  3. When ready, toggle to Live mode in your Stripe Dashboard (top-right switch)
  4. Copy your live API keys (they start with pk_live_ and sk_live_)
  5. Update your credentials here with the live keys
  6. Change the Environment setting to Live
  7. Create a new webhook endpoint with your live signing secret
  8. Process a small test transaction to verify
