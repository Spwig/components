X (Twitter) Setup Instructions

Prerequisites

  • An X (Twitter) account
  • Access to the X Developer Portal
  • A project with OAuth 2.0 enabled

Step 1: Create an X Developer Account

  1. Go to developer.twitter.com
  2. Sign up for a developer account if you don't have one
  3. Complete the application process describing your use case
  4. Wait for approval (usually instant for basic access)

Step 2: Create a Project and App

  1. In the Developer Portal, go to "Projects & Apps"
  2. Click "Create Project"
  3. Name your project and describe its use
  4. Create an app within the project
  5. Select the appropriate environment (Production for live use)

Step 3: Configure OAuth 2.0

  1. Go to your app's "Settings" tab
  2. Under "User authentication settings", click "Set up"
  3. Enable OAuth 2.0
  4. Set the App type to "Web App, Automated App or Bot"
  5. Add the following Callback URL:
{redirect_uri}

Also add a Website URL (your store's domain).

Step 4: Configure App Permissions

Set the following permissions in your app settings:

  • Read and write - Required for posting tweets

Required OAuth 2.0 scopes:

  • tweet.read - Read tweets
  • tweet.write - Post tweets
  • users.read - Read user profile
  • offline.access - Refresh tokens

Step 5: Get Your App Credentials

  1. Go to the "Keys and tokens" tab
  2. Under "OAuth 2.0 Client ID and Client Secret", generate credentials
  3. Copy your Client ID and Client Secret
  4. Add these to your platform's environment variables:
    • TWITTER_CLIENT_ID
    • TWITTER_CLIENT_SECRET

Important: Keep your Client Secret secure and never expose it publicly.

Step 6: Connect Your Account

  1. Click "Authorize" below to start the OAuth flow
  2. Log in with your X account
  3. Grant the requested permissions
  4. You'll be redirected back to configure your account

Note: X posts will be published directly to the account that authorizes the app.
