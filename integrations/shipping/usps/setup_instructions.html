{% trans "Follow these steps to get your USPS API credentials for rate calculation, label generation, and tracking." %}

1

{% trans "Create a USPS Developer Portal Account" %}

{% trans "Register for a free USPS Developer Portal account to access the API." %}

  1. {% trans "Visit the USPS Developer Portal" %}
  2. {% trans "Click 'Sign Up' in the top right corner" %}
  3. {% trans "Complete the registration form with your email and business information" %}
  4. {% trans "Verify your email address by clicking the link sent to your inbox" %}
  5. {% trans "Log in to the Developer Portal" %}
{% trans "Open USPS Developer Portal" %}
2

{% trans "Register Your Application" %}

{% trans "Create an application in the Developer Portal to get your API credentials." %}

  1. {% trans "Log in to the Developer Portal" %}
  2. {% trans "Navigate to 'Apps' in the main menu" %}
  3. {% trans "Click 'Register App' or 'Create New App'" %}
  4. {% trans "Fill in application details:" %}
    • {% trans "App Name: Your store/business name" %}
    • {% trans "Description: Brief description of your shipping integration" %}
    • {% trans "Redirect URI: Leave default if not using OAuth authorization code flow" %}
  5. {% trans "Submit the application registration" %}
{% trans "You may need to wait for approval before your app can access production APIs. Test environment is usually available immediately." %}
3

{% trans "Get Your Consumer Key and Consumer Secret" %}

{% trans "After registering your app, you'll receive OAuth 2.0 credentials." %}

  1. {% trans "Go to 'Apps' section in the Developer Portal" %}
  2. {% trans "Click on your registered application" %}
  3. {% trans "Find the 'Credentials' or 'Keys' section" %}
  4. {% trans "Copy the following values:" %}
    • {% trans "Consumer Key" %} {% trans "(also called Client ID)" %}
    • {% trans "Consumer Secret" %} {% trans "(also called Client Secret)" %}
{% trans "Keep your Consumer Secret secure and never share it publicly. Treat it like a password." %}
4

{% trans "Set Up Payment Account (Required for Label Generation)" %}

{% trans "To generate shipping labels, you need a USPS payment account. This is optional if you only need rate calculation and tracking." %}

  1. {% trans "Choose your payment account type:" %}
    • {% trans "Enterprise Payment System (EPS)" %} - {% trans "For high-volume shippers" %}
    • {% trans "Permit Account" %} - {% trans "For bulk mail" %}
    • {% trans "Postage Meter" %} - {% trans "For PC Postage meters" %}
  2. {% trans "Contact USPS Business Customer Support to set up your payment account" %}
  3. {% trans "Once approved, you'll receive a 10-digit account number" %}
  4. {% trans "Use the Payments API to generate a payment authorization token (valid 60 days)" %}
{% trans "If you only need rate quotes and tracking, you can skip this step. Labels require a valid payment account." %}
{% trans "Learn About Enterprise Payment" %}
5

{% trans "Choose Your Environment" %}

{% trans "USPS provides separate test and production environments." %}

{% trans "Test Environment (TEM)" %}
  • {% trans "Use for development and testing" %}
  • {% trans "No real shipments or charges" %}
  • {% trans "Test tracking numbers available" %}
  • {% trans "Base URL: apis-tem.usps.com" %}
{% trans "Recommended for initial setup and testing" %}
{% trans "Production Environment" %}
  • {% trans "Use for live shipments" %}
  • {% trans "Real postage charges apply" %}
  • {% trans "Requires approved developer account" %}
  • {% trans "Base URL: apis.usps.com" %}
{% trans "Only use after thorough testing in Test environment" %}
6

{% trans "Test Your Connection" %}

{% trans "After entering your credentials in the next step, test the connection to verify everything is set up correctly." %}

  1. {% trans "Enter your Consumer Key and Consumer Secret" %}
  2. {% trans "Optionally enter your Payment Account Number" %}
  3. {% trans "Select your environment (Test or Production)" %}
  4. {% trans "Click 'Test Connection' to verify" %}
{% trans "A successful test means your credentials are valid and you can proceed with configuration." %}
