Setting Up Stripe Terminal

1. Get Your Stripe API Keys

  1. Log in to your Stripe Dashboard
  2. Navigate to Developers → API keys
  3. Copy your Secret key (starts with sk_live_ for production or sk_test_ for testing)
Use test mode keys (sk_test_...) during initial setup. You can switch to live keys when ready for production.

2. Enable Stripe Terminal

  1. In your Stripe Dashboard, go to Terminal → Overview
  2. If prompted, accept the Terminal terms of service
  3. Stripe Terminal is now enabled on your account

3. Create a Location (Optional)

Locations help organize your readers by physical store. You can:

  • Let Spwig create it automatically – Leave the Location ID field blank and we'll create one based on your store settings
  • Use an existing location – Find your Location ID in Terminal → Locations (starts with tml_)

4. Order a Card Reader

Stripe offers several reader options for different use cases:

Reader Best For Connection
Stripe Reader S700 Counter-top retail WiFi/Ethernet
WisePOS E Counter-top retail WiFi/Ethernet
Stripe Reader M2 Mobile/portable Bluetooth

View all Stripe Terminal hardware →

5. Testing Without Hardware

You can test the integration using simulated readers:

  1. Enable "Simulated Reader" in the provider settings
  2. In the POS app, discover and connect to a simulated reader
  3. Use Stripe's test cards to simulate payments
Disable simulated readers before going live to prevent accidental test transactions.
